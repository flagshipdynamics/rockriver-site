Supported File Formats

RockSolid Money™ Support

Home › RockSolid Money › Support › Supported File Formats

RockSolid Money imports the file formats banks, card issuers, and brokerages actually produce:

  • CSV / TSV — the common “download transactions” export. Profiles for major US banks and card issuers ship with the application; unknown layouts go through the guided mapping flow.
  • OFX / QFX — Open Financial Exchange and Quicken's variant of it. These carry bank-assigned transaction IDs, which strengthens duplicate detection.
  • QIF — the legacy Quicken interchange format, which is also a common export path out of Microsoft Money.

Banking, investment, and crypto files

Both editions import everything: checking/savings/card activity, brokerage activity (buys, sells, dividends, interest), and crypto exchange history. Import breadth is never an upsell — the editions differ in reporting, not in what can be read (see Essentials and Pro).

When a file can't be recognized

A file the application cannot confidently interpret is quarantined and explained — it is never half-parsed into something that looks plausible but is wrong. The explanation says what was missing or ambiguous, so you know whether to pick a different export format from your bank or walk through the mapping flow (see When Your Bank Isn't Recognized).

Migrating from Microsoft Money or Quicken

Export from the old application as QIF (or OFX/QFX where offered) and import here. Direct reading of legacy Money/Quicken database files is planned but not available yet.
