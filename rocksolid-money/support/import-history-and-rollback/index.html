Import History and Rollback

RockSolid Money Support

Home › RockSolid Money › Support › Import History and Rollback

Every import commit is recorded as a session: when it ran, what file it read, and exactly what it did — transactions created, updated, linked, and skipped as duplicates. Find it under Data > Import History (Ctrl+H).

Inspecting a session

Select a session to see its details in the Inspector. This is the audit trail for the question “where did these transactions come from?” — and each individual transaction also carries its own import evidence.

Previewing a rollback

Preview Rollback (dry run) shows what undoing a session would do — which transactions would be removed and what would be affected — without changing anything. Read it before rolling back a session you committed weeks ago.

Rolling back

Roll Back undoes the session as a unit. If the rollback would discard work you've done since the import — categorizations, a reconciliation that includes those transactions — the application tells you what's at stake instead of proceeding silently.

What rollback is for

  • You imported a file into the wrong account.
  • You confirmed a mapping and then realized a column was misassigned.
  • You want to re-import the same file under a corrected profile.

Roll the session back, fix the cause, import again. Duplicate detection makes the re-import safe.
