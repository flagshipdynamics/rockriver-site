Backups and Restore

RockSolid Money™ Support

Home › RockSolid Money › Support › Backups and Restore

Your ledger is a file on your machine — which means backing it up is yours to control, and the application makes the state of that duty visible instead of forgettable.

Backing up

Back Up Now lives in the File menu, the file selector's menu, and the backup status menu in the top bar. A backup captures everything: the ledger, imported source files (your import evidence), and import profiles.

The status is always visible

The top bar shows backup state at all times: Backed up 6 hours ago, Backup due, Never backed up, or — in red, the one alarm it earns — Backup failed. A failed backup also lands in the dashboard's Needs Attention panel.

Verifying

Verify Last Backup checks the most recent archive's integrity — that it exists, is complete, and is readable. A backup you've never verified is a hope, not a plan.

Restoring

Restore from Backup… lists your backup history; restoring replaces current data with the archive's contents. Use it after a machine migration or to walk back from a mistake bigger than an import rollback covers.

Backup settings

Backup Settings… configures where backups are written. Point it somewhere that survives your computer — an external drive or a folder synced by a service you already trust.
