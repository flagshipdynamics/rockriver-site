Importing Your First Statement

RockSolid Money™ Support

Home › RockSolid Money › Support › Importing Your First Statement

RockSolid Money never connects to your bank and never asks for banking credentials. You download an export from your bank's website and import the file — the application does the rest, and shows its work.

1. Download an export from your bank

Log in to your bank, card, or brokerage site and look for Download, Export, or Statements & Documents. Choose CSV, OFX/QFX, or QIF (see Supported File Formats). Any date range works — overlapping ranges are safe (see Duplicate Detection).

2. Start the import

Click Import in the top bar, or File > Import File… (Ctrl+I), and choose the downloaded file.

3. Review the interpretation

Before anything is recorded, the import workbench shows you how the file will be read:

  • Recognized bank? Files from known institutions import against a shipped profile — usually nothing to review but the preview.
  • Unrecognized format? The application proposes a column mapping and shows sample rows interpreted under it. Where the file carries a running balance, the sign convention (is this row money in or money out?) is proven arithmetically against that balance rather than guessed. If arithmetic can't settle it, you are asked in plain language. See When Your Bank Isn't Recognized.
  • Which account? Confirm which of your accounts the file belongs to, or create the account right there.

4. Commit

Click Commit to write the transactions. The commit is recorded as an import session you can inspect and — if needed — roll back as a unit (see Import History and Rollback).

After the import

  • Every imported transaction can show the exact source row it came from — select it and use View Import Evidence (see Import Evidence).
  • Imports don't assign categories automatically. Categorize from the register, and see Categories and Payees.
  • Import the next statement whenever you like — the saved profile makes later files from the same institution one-click.
