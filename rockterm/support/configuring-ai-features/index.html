Configuring AI Features

Last updated: April 2026

RockTerm includes an AI assistant powered by Anthropic's Claude that can help you interpret terminal output, troubleshoot errors, and navigate unfamiliar configuration syntax. This article explains what the AI can do, how to enable it, and how to use it effectively.

What AI Can Do

The AI assistant acts as a knowledgeable advisor sitting alongside your terminal session. It can:

  • Interpret errors — explain cryptic error messages from SSH, Linux, Cisco IOS, Junos, and other platforms
  • Suggest commands — recommend the right command or syntax for a task you describe in plain language
  • Explain output — break down complex output such as routing tables, BGP neighbor summaries, ACL hit counts, or systemd journal logs
  • Help with configuration syntax — assist with config blocks for Cisco IOS/IOS-XE/NX-OS, Arista EOS, Juniper Junos, Palo Alto PAN-OS, Linux (iptables, netplan, systemd units), and more
  • Provide context — explain what a protocol does, how a feature works, or what the implications of a configuration change might be

Enabling AI Features

AI features are disabled by default. To enable them:

  1. Open Settings (Ctrl+,)
  2. Navigate to the AI tab
  3. Toggle Enable AI Assistant to on
  4. Enter your Anthropic API key (see AI API Key Setup for details)
  5. Click Save

Once enabled, the AI button appears in the toolbar and the keyboard shortcut becomes active.

How to Use the AI Assistant

There are two ways to invoke the AI assistant:

  • Keyboard shortcut: Press Ctrl+Shift+A to open the AI panel
  • Toolbar button: Click the AI icon in the terminal toolbar

When invoked, the AI panel slides open alongside the terminal. Type your question or prompt in the input field and press Enter. The AI automatically sees the most recent terminal output (approximately the last 200 lines of the active session's scrollback buffer) as context, so you do not need to copy and paste terminal content into your prompt.

The AI responds in the panel with formatted text, including code blocks you can copy with a single click. The terminal session continues running normally while the AI panel is open — you can type commands in the terminal and interact with the AI simultaneously.

Example Prompts

Here are some practical prompts to get you started:

Error Interpretation

What does this error mean?
Why am I getting "Permission denied (publickey)" when connecting?
Explain the BGP notification message in the output above.

Command Assistance

How do I configure OSPF area 0 on this interface?
What's the command to show the ARP table on NX-OS?
How do I create a static route to 10.10.0.0/16 via 192.168.1.1 on IOS-XE?

Output Explanation

Explain this routing table.
Are there any problems visible in this BGP summary?
What do these syslog messages indicate?
Summarize the interface status output above.

Configuration Help

Generate a config for an HSRP group on VLAN 100 with priority 110.
Write an iptables rule to allow inbound TCP 443 and drop everything else.
Help me write a netplan config for a bonded interface with LACP.

Limitations

The AI assistant is a powerful tool, but it is important to understand its boundaries:

  • Advisor, not operator. The AI provides suggestions and explanations. It cannot execute commands on the remote host. You must review every suggestion and decide whether to apply it.
  • Cannot execute commands. The AI has no ability to type into the terminal, send keystrokes, or modify the SSH session in any way. It is read-only with respect to the terminal.
  • Always verify before executing. AI-generated commands and configurations should be treated like advice from a colleague: helpful, but you should verify correctness against vendor documentation and test in a non-production environment when possible.
  • Context window. The AI sees only the most recent terminal output (approximately 200 lines). If the relevant output has scrolled beyond this window, you can select and copy the text into your prompt manually.
  • No persistent memory. Each invocation is independent. The AI does not remember previous questions or answers from earlier in your session. If you need to reference a prior conversation, include the relevant context in your new prompt.
