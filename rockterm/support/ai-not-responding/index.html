AI Not Responding

Last updated: April 2026

If RockTerm's AI features are not responding, returning errors, or timing out, work through the following troubleshooting steps in order. Most issues can be resolved quickly.

1. Check Your Internet Connection

RockTerm's AI features require an active internet connection to communicate with the Anthropic API over HTTPS.

  • Verify that your machine can reach external websites.
  • Confirm that HTTPS traffic to api.anthropic.com is not blocked. You can test this by running:
    curl https://api.anthropic.com
  • If you are on a restricted network, see the firewall and proxy section below.

2. Verify Your API Key

An invalid, expired, or revoked API key will prevent AI features from functioning.

  • Open Settings > AI in RockTerm and confirm that an API key is entered.
  • Check that the key has not been accidentally truncated or modified.
  • Sign in to console.anthropic.com and verify that the key is still active.
  • If you are unsure whether the key is valid, generate a new API key from the Anthropic console, enter it in RockTerm, and try again.

3. Check for API Rate Limits or Billing Issues

Anthropic enforces rate limits and requires an active billing arrangement.

  • Visit the Anthropic dashboard and check for any alerts or warnings on your account.
  • Verify that your account has a valid payment method on file and that you have not exceeded your spending limit.
  • If you have hit a rate limit, wait a few minutes and try again. Rate limits typically reset after a short period.

4. Corporate Firewall or Proxy

If you are on a corporate or institutional network, security policies may block outbound HTTPS requests to api.anthropic.com.

  • Contact your network administrator to confirm that traffic to api.anthropic.com (port 443) is permitted.
  • If your network requires a proxy, ensure that your system's proxy settings are configured correctly. RockTerm respects the system-level proxy configuration.
  • Some deep packet inspection (DPI) systems can interfere with API traffic. If you suspect this is the case, work with your IT department to whitelist the Anthropic API endpoint.

5. Check for an Anthropic API Outage

On rare occasions, the Anthropic API may experience service disruptions.

  • Visit status.anthropic.com to check for any ongoing incidents or maintenance windows.
  • If there is a reported outage, wait for Anthropic to resolve the issue and try again later.

6. Confirm AI Features Are Enabled

AI features in RockTerm are disabled by default and must be turned on before use.

  • Open Settings > AI in RockTerm.
  • Ensure the Enable AI Features toggle is switched on.
  • If you recently updated RockTerm, verify that your AI settings were preserved. In some cases, a major update may reset preferences to their defaults.
