Using Connection Check

RockWiFi™ Pro Support

Home › RockWiFi Pro › Support › Using Connection Check

Connection Check is a full-stack connectivity diagnostic that tests each network layer in sequence to pinpoint exactly where a problem lies. Instead of guessing whether the issue is Wi-Fi, DNS, or something upstream, Connection Check walks through every layer and reports what works, what doesn't, and why.

What It Tests

Connection Check runs the following tests in order, stopping at the first failure to identify the root cause:

  1. Wi-Fi Adapter — Verifies a wireless adapter is present and connected to a network.
  2. Gateway Reachability — Pings the default gateway to confirm local network connectivity.
  3. Public Internet — Tests connectivity to the public internet beyond the local network.
  4. DNS Resolution — Tests your system DNS resolver, plus Cloudflare (1.1.1.1) and Google (8.8.8.8) as independent references.
  5. HTTPS Test — Confirms that secure web traffic can complete a full TLS handshake.
  6. VPN Detection — Identifies active VPN connections that may be affecting routing or DNS behavior.

Wi-Fi Details

When a Wi-Fi connection is active, Connection Check displays detailed information about your current association:

  • SSID and BSSID — The network name and the specific access point you are connected to.
  • Signal strength — Current signal level in dBm.
  • Wi-Fi standard — Shown as a friendly name (Wi-Fi 7, Wi-Fi 6, Wi-Fi 5, Wi-Fi 4).
  • Channel and width — The channel number and bandwidth (20/40/80/160/320 MHz).
  • Max rate — The maximum PHY rate negotiated with the AP.
  • Station count — Number of clients associated to the same AP (when reported by the AP).
  • Airtime utilization — Percentage of channel airtime in use.
  • Beacon interval — Time between AP beacons in milliseconds.
  • Short GI — Whether short guard interval is in use.
  • Country code — The regulatory domain reported by the AP.
  • MLO status — Multi-Link Operation status for Wi-Fi 7 connections.
  • Authentication — RSN-level authentication detail (e.g., SAE, RSN-PSK, 802.1X).
  • Cipher — The encryption cipher in use (e.g., CCMP, GCMP-256).
  • Security rating — An overall assessment of the connection's security posture.

Reading the Diagnosis

After all tests complete, Connection Check provides a structured diagnosis:

  • Likely issue — A plain-language description of the most probable problem.
  • Confidence level — How certain the diagnosis is based on the evidence gathered.
  • Evidence list — The specific test results that support the diagnosis.
  • Next steps — Recommended actions to resolve the issue.
  • Ruled-out causes — Problems that were eliminated by passing tests, so you don't waste time investigating them.

BSS-Informed Next Steps

Connection Check uses the BSS (Basic Service Set) data from your current association to provide targeted recommendations:

  • Legacy standard warning — If you are connected using Wi-Fi 4 or older when the AP supports newer standards, Connection Check recommends upgrading your adapter.
  • Narrow channel suggestion — If the connection is using a 20 MHz channel when wider channels are available, it suggests checking AP configuration.
  • High utilization alert — If airtime utilization is high, it recommends checking for congestion or moving to a less busy channel or band.

VPN Detection

When a VPN is active, Connection Check identifies it and reports:

  • Adapter name — The virtual network adapter created by the VPN client.
  • Routing mode — Whether the VPN is using split tunnel (only some traffic goes through the VPN) or full tunnel (all traffic routes through the VPN).
  • DNS changes — Whether the VPN has overridden your DNS servers, which can affect resolution behavior.

Tips

  • Run Connection Check before and after making changes to verify improvement.
  • Use the Re-run button for a quick retest without navigating away.
  • If Connection Check passes all layers but you still have issues, the problem may be application-specific or server-side rather than network-related.
